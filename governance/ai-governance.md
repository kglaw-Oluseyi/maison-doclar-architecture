# AI Governance Charter v0.1 (re-charter, ratified in principle)
Govern, don't proxy. The governance service owns: policies, prompt registry, model registry, budget management, fallback routing policy, evaluation, cost analytics, PII controls, safety policies, provider compliance. It is **never in any execution path**; products call providers directly under registered configurations. Migration is registration, not rewiring: each of the ~15 current call sites registers its model/prompt/budget profile. Donor knowledge: maison-doclar-intelligence's app-registry concept; Gateway's key-management (one-time reveal) pattern.
