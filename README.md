# Maison Doclar — Constitutional Repository

This is the constitutional repository of Maison Doclar. It is the single versioned home for the assets that outlive any product, repository, or generation of software:

- `/constitution` — MPS (Maison Protocol System), MXS (Maison Experience System), MKC (Maison Knowledge Constitution). Versioned constitutional assets. Every event, recommendation, and protocol decision pins the versions under which it was made.
- `/capabilities` — the capability catalogue (C1–C24) and the realization register. **Maison owns capabilities; systems realize them.**
- `/knowledge` — the Knowledge Preservation Matrix (v1.0 + v1.1 addendum), Architectural Fossils, Negative Knowledge, and Human Knowledge Assets registers.
- `/decisions` — the decision register: the seed corpus of Decision Intelligence (C24). Every constitutional decision is recorded with rationale, alternatives, and evidence.
- `/architecture` — the Strategic Architecture Program deliverables and engineering standards. Architecture is one constitutional artifact among several, not the whole.
- `/governance` — repository classifications (versioned constitutional state), AI governance charter, repository strategy.
- `/contracts` — the Advisory Contract and pipeline contracts between the Execution Plane, Intelligence Plane, and external systems.

## Governing principles (ratified)
1. The asset hierarchy is **Knowledge → Capability → Service → Experience**. Repositories are temporary realizations.
2. The Platform runs two planes: an event-scoped **Execution Plane** that owns operational truth, and an **Intelligence Plane** that reasons over the governed **Knowledge Store** and returns advice only.
3. Execution never queries Intelligence; it requests advice and always records its decision. Advice degrades to absence, never to failure.
4. Every knowledge artifact is invalid unless MKC accepts it. MKC preserves failures with the same rigor as successes.
5. No repository transition (read-only, archive) occurs without explicit named approval, an execution plan, and a Decision Intelligence record.
