# Negative Knowledge Register
Failures preserved with the same rigor as successes (MKC Article 7). Format: do-not-repeat · reason · evidence.

1. **Federated microservices at sole-operator scale.** ~20 services exceeded operational capacity regardless of architectural cleanliness. Evidence: the estate's revealed re-consolidation into Aperture (Gen-2 → Gen-3).
2. **Cross-event operational joins.** Contamination occurred. Evidence: the event-scope verify script and its migration context. Cross-event flows exist only via declared Intelligence pipelines.
3. **"No response" as an RSVP state.** It hid escalation work inside a status. Evidence: `retire_rsvp_no_response`.
4. **Mandatory AI execution proxy.** Fifteen call sites bypassed it; centralised execution added latency and coupling without governance value. Evidence: dependency scan, 11 Jul 2026. Replacement: govern, don't proxy.
5. **Per-application guest models.** Eight divergent copies bred the class of data bugs that consumed the Alakija preparation period. Evidence: schema scan across nucleus/aperture/pwa/facegate/ESLA/remote-view/giftpack/relay.
6. **Provider dependence without an exit.** The Cloudinary→B2 migration cost real effort across the estate. Rule: storage/messaging/AI providers sit behind owned adapters.
7. **Unversioned architecture.** Three conflicting "source of truth" documents; the Event OS specification lived only in conversation history. Evidence: Deliverable 01 F7. Remedy: this repository.
